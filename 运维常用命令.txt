##docker/overlay2 磁盘空间占满解决方案；需要先停止容器；迁移/var/lib/docker/overlay2 到挂载大磁盘的目录下面；再使用软连接指向 /var/lib/docker/overlay2
# 1. 停止 Docker
systemctl stop docker

# 2. 挂载大容量磁盘（如 /dev/sdb1 到 /data/docker）；如果已经有挂载到大目录的磁盘；这步可以忽略
mount /dev/sdb1 /data/docker

# 3. 迁移 overlay2 目录
mv /var/lib/docker/overlay2/* /data/docker/
rm -rf /var/lib/docker/overlay2
ln -s /data/docker /var/lib/docker/overlay2

# 4. 启动 Docker
systemctl start docker

## 补全docker-compose.yaml的network配置

	# 一键补全 docker-compose.yml 的 networks 配置（保留原有内容，追加配置）
sed -i '/^networks:$/a\\  default:\\n    driver: bridge\\n    ipam:\\n      config:\\n        - subnet: 10.10.0.0/16\\n          gateway: 10.10.0.1' docker-compose.yml


##Harbor 的 docker-compose.yml 中服务通常需要统一的自定义网络（而非 default），执行以下命令可创建名为 harbor-network 的自定义网络，并让所有服务加入
# 第一步：删除原有空的 networks 配置，重建自定义网络
sed -i '/^networks:$/,/^[^ ]/d' docker-compose.yml  # 删除原有空的 networks 块
cat >> docker-compose.yml << EOF
networks:
  harbor-network:
    driver: bridge
    ipam:
      config:
        - subnet: 10.10.0.0/16
          gateway: 10.10.0.1
EOF

# 第二步：将所有服务的 networks 改为 harbor-network
sed -i '/^    networks:$/c\\    networks:\\n      - harbor-network' docker-compose.yml
 
## sed命令
删除含指定关键词的行	sed -i '/关键词/d' 文件名
删除空行	sed -i '/^$/d' 文件名
删除指定行号	sed -i 'Nd' 文件名
删除匹配行 + 后续行	sed -i '/关键词/,+Nd' 文件名
删除指定字符串（保留行）	sed -i 's/字符串//g' 文件名

总结
核心问题：172.30.0.141 上的 Docker 网桥创建了错误路由，把所有 10 段 IP 的回包导向了 Docker 虚拟网卡，导致回程包丢失；
临时修复：添加指向 10.11.3.0/24 的静态路由，指定走物理网卡和正确网关；
永久修复：要么修正 Docker 网络配置，要么添加永久静态路由。
你先执行临时修复命令，验证 ping 是否能通，再做永久配置即可
